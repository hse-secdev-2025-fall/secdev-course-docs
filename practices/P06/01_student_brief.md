# Инструкция для студента (P06 — Secure Coding)

## 1) Ветка и структура
1. От `main` создайте ветку `p06-secure-coding`.
2. Работайте в `src/...`; тесты в `tests/...`.

## 2) Реализуйте ≥3 контроли из списка
- Проверьте, чтобы они были **измеримыми** и **проверяемыми** (линтер, тест, контракт, конфиг).

## 3) Тесты (обязательно негативные)
- На каждый контроль — ≥1 негативный тест (большой файл, неверная сигнатура, path traversal, float‑погрешность, отсутствие таймаутов и т.п.).

## 4) Связи с предыдущими практиками
- В описании PR укажите ссылки на `NFR-xx` (P03) и `F#/R#` (P04), если контроль закрывает риск/угрозу.

## 5) PR и ревью
- PR по шаблону, CI зелёный, артефакты тестов приложены/загружены.
